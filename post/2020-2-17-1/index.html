Paper Note - 2020/2/17

(ICDE 2018) Price-and-Time-Aware Dynamic Ridesharing.

Tong Yu
Feb 17, 2020 5 min read Paper Note - ICDE

1. Scenario

  1. Ridesharing refers to a transportation scenario where travellers with similar itineraries and time schedules share a vehicle for a trip and split the travel costs such as fuel, tolls, and parking fees.
  2. When travellers need to travel to some destination, they often have several options to choose among. In such a setting, the cost and travel time of an option are important decision criteria.
  3. In the paper, it’s assumed that the destination of a vehicle is not limited and that it can accommodate any number of rider groups during a trip as long as it satisfies a capacity constraint.

2. Motivation

Given a set of vehicles $C$ in the road network $G$ at a specific time and a real-time trip request set $S_{R}$ , dynamic ridesharing finds, for each trip request $R \in S_{R}$ , all qualified and non-dominated results $<c, time, price>$ .

3. Preliminaries

3.1. Symbols

3.2. Definition

  1. Ridesharing Request : A ridesharing request $R=<s,d,n,w,\epsilon>$ on a road network $G$ is defined by a start location $s$ , a destination location $d$ , a number $n$ of riders, a maximal waiting time $w$ , and a service constraint $\epsilon$ .
  2. Vehicle Trip Schedule : Each trip schedule $tr=<o_{1},o_{2},\cdots,o_{k}>$ contains a sequence of locations, where $o_{1}$ is the current location $c.l$ of vehicle $c$ , and $o_{i}$ $(2 \leq i \leq k)$ denotes the start location or destination of an unfinished ridesharing request.
  3. Schedule Constraint : A valid trip schedule needs to satisfy the four following conditions
    1. Capacity constraint
    2. Point order
    3. Waiting time constraint
    4. Service constraint
  4. Price Model : For a ridesharing request $R=<s,d,w,n,\epsilon>$ , assume that $tr_{i}$ is the current trip schedule of a vehicle $c$ and $tr_{i}$ is the new trip schedule after inserting $R$ into $tr_{i}$ , the price is computed as $f_{n} \times (dist_{tr_{j}}-dist_{tr_{i}}+dist(s,d))$ , where $f_n$ is price ratio that depends on the number $n$ of riders.

4. Method

4.1. Index Structures

4.1.1. Road Network Indexing

  1. If an edge $e=(u,v)$ belongs to more than one grid cell, we call $u$ and $v$ border vertices.
  2. To estimate the shortest path distance precisely, we maintain a matrix $M$ that stores the lower bound distance for every grid cell pair.
  3. Each grid cell maintains five lists :
    1. border vertex list : border vertices belonging to the grid cell
    2. vertex list : vertices belonging to the grid cell, as each vertex $v$ is associated with its shortest path distances to the border vertices ($BV$) of the grid cell ($\{ < u,dist(v,u) > | u \in BV \}$) and is also associated with $\min \{ dist(v,u) | u \in BV \} $ (denoted as $v.min$ )
    3. grid cell list : other grid cells sorted in ascending order of the travel time from these grid cells to the grid cell.
    4. empty vehicle list : vehicles with empty ridesharing requests in $g_{i}$
    5. non-empty vehicle list : vehicles $c_j$ with a non-empty set of ridesharing requests, i.e., vehicles $c_j$ are currently located in $g_j$ or are scheduled to enter $g_{i}$
  4. Lower Bound Distance : Given a vertex $u$ in a grid cell $g_i$ and a vertex $v$ in a grid cell $g_j$ , if $g_i \neq g_j$ , then the lower bound distance $ldist(u, v) = D_{ij} + u.min + v.min$ ; if $gi = gj$ , then $ldist(u, v) = 0$ .
  5. Upper Bound Distance : if $g_i \neq g_j$ , then the upper bound distance $udist(u, v) = D_{ij} + dist(u, x_{ij}) + dist(v, y_{ij})$ ; if $g_i = g_j$ , then $udist(u, v) = \min \{dist(u, b) + dist(v, b) | b \in BV \}$ .

4.1.2. Index of Vehicle Trip Schedules

  1. Representation of each vehicle $c$ :
    1. the unique identifier $c.ID$ of the vehicle
    2. the current location $c.l$ of the vehicle
    3. the set $c.S$ of unfinished ridesharing requests assigned to it, sorted in ascending order of their timestamps
    4. the set $c.S_{tr}$ of all valid vehicle trip schedules
  2. The set $c.S_{tr}$ of all valid vehicle trip schedules can be managed by a kinetic tree . Three variables are added to each node $o_x$ of the kinetic tree :
    1. a current vehicle’s capacity
    2. the minimal detour distance allowed
    3. the trip distance disttr from $c$’s current location to $o_x$
  3. The following global information of each grid cell $g_j$ is maintained to facilitate the pruning of a whole grid cell :
    1. the maximum capacity : $\max \{ o_x.capacity | <o_x, o_y> \in g_j \}$
    2. the maximal detour allowed : $\max \{ o_x.detour | <o_x, o_y> \in g_j \}$
    3. the minimal trip distance : $dist_{tr}=\min \{ o_x.dist_{tr} | <o_x, o_y> \in g_j \}$
    4. the maximal shortest path distance : $dist=\max \{ dist(o_x,o_y) | <o_x,o_y> \in g_j \}$

4.2. Pruning Heuristics

Main Idea : Based on the designed road network and vehicle indexes, we use the lower and upper bound distances instead of the actual shortest path distances to estimate pick-up time and price for filtering.

4.2.1. Pruning Empty Vehicles

  1. Given a current valid result $r_i$, we try to derive :
    1. the pruning conditions implying that the estimated result $r_j$ (i.e., the lower bound values) returned by $c_j$ is dominated by $r_i$
    2. the pruning conditions implying that $r_j$ (i.e., the upper bound values) returned by $c_j$ dominates $r_i$
  2. Based on the main idea, we can filter out invalid vehicles and grids.
